## Step 3: FL 算法模块完成

### 创建的文件 (fl_space/fl/)
- `core.py` — 四个核心抽象接口：ClientSelector, LocalTrainer, Aggregator, Evaluator + 数据结构（ClientState, ClientUpdate, FLRoundResult）
- `models.py` — MLP / SimpleCNN 示例模型 + 工厂函数 + 自定义模型注册机制
- `fedavg.py` — FedAvg 算法组件：RandomSelector + FixedEpochTrainer + SyncWeightedAggregator + StandardEvaluator + create_fedavg_components()
- `fedprox.py` — FedProx 算法组件：复用 selector/aggregator/evaluator，仅替换 ProximalTrainer（含proximal term μ·||w-w_global||²）
- `fedbuff.py` — FedBuff 异步算法：AsyncSelector + AsyncTrainer + BufferAggregator（FIFO缓冲区聚合，支持staleness降权）
- `scheduler.py` — 通信调度器：读取模拟器接触矩阵 → 通信窗口提取 → 连通性查询，完全独立于FL算法
- `server.py` — FLServer 编排器 + FLConfig 配置类：支持 run_sync()（FedAvg/FedProx）和 run_async()（FedBuff）双模式
- `config.py` — 预设配置：算法预设 + 规模预设（small/medium/large）+ 数据集预设（MNIST/CIFAR10）+ 6个组合预设
- `runner.py` — FLRunner 一键运行器：数据加载（IID/non-IID Dirichlet）→ 模型创建 → 训练执行
- `__init__.py` — 完整重导出 + __all__ 列表

### 设计要点
- **四组件解耦**：每个FL算法 = ClientSelector + LocalTrainer + Aggregator + Evaluator，组件可独立替换
- **算法间复用**：FedProx 只替换 trainer，FedBuff 替换 selector + aggregator，最大化代码复用
- **调度器独立**：完全遵循导师建议 — 通信判断与算法逻辑分离，接受OrbitSimulator引用输出连通性
- **遵循CODING_STANDARDS.md**：ruff check + ruff format 均通过，Python 3.9+ 类型注解，NumPy风格中文docstring
- **导入验证通过**：`import fl_space.fl` 成功，预设列表正常

### 五个子命令
| 命令 | 功能 | 典型用法 |
|------|------|---------|
| `fl-space simulate` | 轨道接触模拟 + 调参 | `--sats 10 --stations 5 --hours 24 --backend kepler` |
| `fl-space train` | FL 训练实验 | `--algo fedprox --dataset cifar10 --rounds 100 --device cuda` |
| `fl-space list` | 查看内置资源 | `presets|models|satellites|experiments` |
| `fl-space export` | 导出模拟结果JSON | `--body mars --sats 5 --output mars.json` |
| `fl-space info` | 系统环境信息 | (无参数) |

### 调参覆盖
- **simulate**: 卫星数、地面站数、时长、后端、轨道高度、倾角、分布策略、时间粒度
- **train**: 算法(fedavg/fedprox/fedbuff)、数据集、规模、轮次、epoch、学习率、batch size、μ(FedProx)、缓冲K(FedBuff)、staleness降权、IID/non-IID、α(Dirichlet)
- **全局**: --output JSON导出、--quiet 安静模式
